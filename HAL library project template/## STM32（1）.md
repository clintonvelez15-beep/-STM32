## STM32（1）

6.1HAL 库工程模板

采用STM32F103C8T6最小开发板，软件STM32cubeIDE进行开发
接下来先讲清楚STM32cubeIDE的基本工作框架，直接给出对应代码讲清楚：

针对**STM32 微控制器的 HAL 库工程模板**

#### 头文件

```
c
#include "main.h"
/* USER CODE BEGIN Includes */
/* USER CODE END Includes */
```

引入 `main.h`，并且允许用户添加自己的头文件

### `main()` 函数

```
c
int main(void)
{
  /* USER CODE BEGIN 1 */
  /* USER CODE END 1 */

  HAL_Init();
  /* USER CODE BEGIN Init */
  /* USER CODE END Init */

  SystemClock_Config();
  /* USER CODE BEGIN SysInit */
  /* USER CODE END SysInit */

  /* Initialize all configured peripherals */
  /* USER CODE BEGIN 2 */
  /* USER CODE END 2 */

  while (1)
  {
    /* USER CODE END WHILE */
    /* USER CODE BEGIN 3 */
  }
  /* USER CODE END 3 */
}
```

- `main.h`
  该头文件包含芯片型号对应的 HAL 库头文件以及工程中定义的外设句柄、引脚宏等。
- **`HAL_Init()`**
  初始化 HAL 库：配置 Flash 预取、SysTick 定时器（作为 HAL 的时基）、中断优先级分组等。
- **`SystemClock_Config()`**
  调用时钟配置函数（见下文），设置系统时钟、总线分频等。
- **`USER CODE BEGIN 2`**
  在这里初始化外设（如 GPIO、UART、ADC 等），是用户添加初始化代码的主要位置。
- **`while(1)` **主循环
  `USER CODE BEGIN 3` 内编写循环执行的应用逻辑。

### 系统时钟配置

```
c
void SystemClock_Config(void)
{
  RCC_OscInitTypeDef RCC_OscInitStruct = {0};
  RCC_ClkInitTypeDef RCC_ClkInitStruct = {0};

  RCC_OscInitStruct.OscillatorType = RCC_OSCILLATORTYPE_HSI;
  RCC_OscInitStruct.HSIState = RCC_HSI_ON;
  RCC_OscInitStruct.HSICalibrationValue = RCC_HSICALIBRATION_DEFAULT;
  RCC_OscInitStruct.PLL.PLLState = RCC_PLL_NONE;
  if (HAL_RCC_OscConfig(&RCC_OscInitStruct) != HAL_OK)
  {
    Error_Handler();
  }

  RCC_ClkInitStruct.ClockType = RCC_CLOCKTYPE_HCLK|RCC_CLOCKTYPE_SYSCLK
                              |RCC_CLOCKTYPE_PCLK1|RCC_CLOCKTYPE_PCLK2;
  RCC_ClkInitStruct.SYSCLKSource = RCC_SYSCLKSOURCE_HSI;
  RCC_ClkInitStruct.AHBCLKDivider = RCC_SYSCLK_DIV1;
  RCC_ClkInitStruct.APB1CLKDivider = RCC_HCLK_DIV1;
  RCC_ClkInitStruct.APB2CLKDivider = RCC_HCLK_DIV1;

  if (HAL_RCC_ClockConfig(&RCC_ClkInitStruct, FLASH_LATENCY_0) != HAL_OK)
  {
    Error_Handler();
  }
}
```

配置系统时钟源和总线分频，配置是：

* **时钟源** ：内部 HSI（8MHz RC 振荡器），不使用 PLL（PLLState = RCC_PLL_NONE）
* **系统时钟 SYSCLK** = 8MHz
* **AHB 总线 HCLK** = SYSCLK / 1 = 8MHz
* **APB1 总线 PCLK1** = HCLK / 1 = 8MHz（最大 36MHz，所以没问题）
* **APB2 总线 PCLK2** = HCLK / 1 = 8MHz（GPIOA 就挂在这条线上）

这意味着单片机正以 8MHz 的速度运行，执行指令很快，但点灯用人眼感觉不到区别。

### GPIO 初始化函数

```
c
static void MX_GPIO_Init(void) {
  GPIO_InitTypeDef GPIO_InitStruct = {0};

  __HAL_RCC_GPIOA_CLK_ENABLE();   // 开启 GPIOA 时钟

  HAL_GPIO_WritePin(GPIOA, GPIO_PIN_1|GPIO_PIN_4|GPIO_PIN_7, GPIO_PIN_RESET);

  GPIO_InitStruct.Pin = GPIO_PIN_1|GPIO_PIN_4|GPIO_PIN_7;
  GPIO_InitStruct.Mode = GPIO_MODE_OUTPUT_PP;   // 推挽输出
  GPIO_InitStruct.Pull = GPIO_NOPULL;           // 无上下拉
  GPIO_InitStruct.Speed = GPIO_SPEED_FREQ_LOW;  // 低速
  HAL_GPIO_Init(GPIOA, &GPIO_InitStruct);
}
```

该部分允许在IDE内进行图形化设置以实现。图形化设置完成后，代码会自动帮助生成对应内容。此处设定启用PA1、PA4、PA7引脚作为示例。

* **开时钟** ：`__HAL_RCC_GPIOA_CLK_ENABLE()` 展开后就是操作 `RCC->APB2ENR` 的第 2 位，和我们寄存器点灯的第一步一样。
* **设置初始电平** ：先让 PA1、PA4、PA7 输出低电平（复位状态）。
* **配置模式** ：三个引脚都设为推挽输出、无上下拉、低速。`HAL_GPIO_Init` 会根据这个结构体去写 CRL/CRH 寄存器。

### 错误处理函数

```
c
void Error_Handler(void) {
  __disable_irq();
  while(1) {}
}
```

如果初始化（时钟、外设）失败，程序会跳到这里，关所有中断，然后死循环。这是嵌入式系统的“蓝屏”，告诉你系统跑不下去了。

* 首先 `__disable_irq()` 关闭全局中断，防止中断干扰。
* 然后进入死循环，等待看门狗复位或用户调试。
* 用户可以在此添加自己的错误上报代码（如点亮 LED、打印日志等），但需写在 `USER CODE BEGIN Error_Handler_Debug` 区域内。

### 断言函数

```
c
#ifdef USE_FULL_ASSERT
void assert_failed(uint8_t *file, uint32_t line) {
  // 用户可以加 printf 打印错误位置
}
```

HAL 库的一些函数会用 `assert_param` 检查参数是否合法。如果参数非法，就调这个函数。通常用于调试，发布时可以关掉。

以上这个 `main.c` 文件便是 STM32 工程的基础框架，而用户在对应 `USER CODE` 区域添加自己的代码。
